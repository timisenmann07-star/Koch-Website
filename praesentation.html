Culinary Kitchen - BK2 Projektpräsentation

Folie 1: Projektübersicht

Was ist Culinary Kitchen?

Eine interaktive Rezept-Website, die es Benutzern ermöglicht:

  • Rezepte zu durchsuchen und anzusehen
  • Favoriten zu speichern (mit Firebase)
  • Sich anzumelden und zu registrieren
  • Neue Rezepte einzureichen

Technologien: HTML, CSS, JavaScript, Firebase

Folie 2: Projektstruktur

Zeige die Ordnerstruktur im VS Code:

Folie 5: JavaScript - Firebase Integration

Firebase-Konfiguration:

// Firebase initialisieren
const firebaseConfig = {
apiKey: "...",
authDomain: "...",
projectId: "koch-website"
};
firebase.initializeApp(firebaseConfig);
const auth = firebase.auth();
const db = firebase.firestore();

Warum Firebase? Einfache Authentifizierung und Datenbank ohne eigenen Server

Folie 6: Authentifizierung

Login-Funktion:

async function login(email, password) {
try {
const userCredential = await auth.signInWithEmailAndPassword(email, password);
alert('Login erfolgreich!');
window.location.href = 'index.html';
return userCredential.user;
} catch (error) {
alert('Login fehlgeschlagen.');
}
}

async/await: Wartet auf Firebase-Antwort ohne Seite zu blockieren

Folie 7: Favoriten-System

Favoriten hinzufügen:

async function addToFavorites(recipeId, recipeName) {
const user = auth.currentUser;
if (!user) {
alert('Bitte anmelden!');
return;
}

// In Firebase speichern
await db.collection('favorites')
.doc(user.uid)
.collection('recipes')
.doc(recipeId)
.set({ recipeId, recipeName });
}

Datenstruktur: favorites/{userId}/recipes/{recipeId}

Folie 8: Herausforderungen

  • Responsive Design: Website auf verschiedenen Bildschirmgrößen
  • Asynchrone Programmierung: Firebase-Aufrufe warten
  • Benutzer-Status: UI je nach Login-Status aktualisieren
  • Datenpersistenz: Favoriten über Sessions speichern

Folie 10: Lessons Learned

  • Planung ist wichtig - klare Struktur von Anfang an
  • Modularer Code (separate JS-Dateien) ist wartbar
  • Firebase vereinfacht Backend-Aufgaben
  • Testen auf verschiedenen Geräten nicht vergessen
  • Dokumentation hilft beim Erklären und Debuggen
